# Plan to Fix Dark Mode in MauiAppPublisher

## Problem Analysis
The dark theme implementation in this Avalonia-based application has issues where certain UI elements remain white or light-colored in dark mode. Analysis of the codebase reveals:

- Theme switching is handled via MainViewModel.cs, setting `Application.Current.RequestedThemeVariant` to `ThemeVariant.Dark` when "Dark" is selected.
- Styles in MainView.axaml define `:dark` pseudo-class overrides for most elements (cards, buttons, text, etc.).
- However, the outermost Border element in MainView.axaml has a fixed dark background (`#0F172A`), causing the padding area around the main content to always appear dark, which may not align with light mode expectations.
- The main Grid has themed backgrounds, but the outer Border overrides this visually.

## Proposed Fixes

1. **Theme the Outer Border Background**
   - Add a class `window-background` to the outermost Border in MainView.axaml.
   - Define styles for `Border.window-background` (light: `#EEF4F8`) and `Border.window-background:dark` (dark: `#0F172A`) to match the main Grid's background.
   - This ensures the entire window background changes appropriately with the theme.

2. **Verify and Fix Missing Dark Styles**
   - Check for any UI elements without `:dark` style overrides (e.g., status message Border has fixed dark colors, which may be intentional).
   - Ensure all Border.card, Button, and other themed elements have proper dark variants.

3. **Test Theme Switching**
   - Confirm the ComboBox in MainView.axaml correctly binds to the Theme property.
   - Verify that RequestedThemeVariant changes propagate to all styled elements.

4. **Additional Improvements**
   - If the status message area should also theme, add classes and dark styles for that Border.
   - Ensure no hardcoded colors remain that should be themed.

## Implementation Steps
1. Edit MainView.axaml to add class to outer Border and define new styles.
2. Test the application in both light and dark modes.
3. Make any additional adjustments if other elements are found to not theme properly.

## Verification
- Run the app and switch to dark mode via the theme ComboBox.
- Confirm window background, card backgrounds, button backgrounds, and text colors change to dark variants.
- Ensure no elements remain white/light in dark mode.
