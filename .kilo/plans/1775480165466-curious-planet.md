# Fix Dark Theme Issue in Avalonia App

## Problem Analysis

The app has two issues in dark mode:
1. **UI stays light** - The theme doesn't change to dark when dark mode is selected
2. **Invisible controls** - Buttons, labels, and dropdown icons become invisible in dark mode

### Current Implementation
- App.axaml: Uses `<FluentTheme />` with `RequestedThemeVariant="Default"`
- MainViewModel: Has a theme selector ("System", "Light", "Dark") that sets `Application.Current.RequestedThemeVariant` when changed
- MainView.axaml: Contains extensive custom styles with `:dark` pseudo-class selectors for theming

### Root Cause
The theme switching logic in `MainViewModel.OnThemeChanged` (lines 74-87) sets `app.RequestedThemeVariant`, but there's a critical issue: **The MainView is styled at the UserControl level with `UserControl.Styles`, not at the Application level.**

When theme changes, the Application-level FluentTheme should apply dark variants to built-in controls, but:
1. The custom styles with `:dark` selectors in MainView.axaml may not be properly applied
2. The FluentTheme's built-in dark variants might not be applied correctly
3. The dropdown icons (DropDownGlyph) and other themed elements might be missing dark variants

## Proposed Solution

1. **Debug the theme change mechanism**
   - Add logging to verify `OnThemeChanged` is called and `RequestedThemeVariant` changes
   - Verify `Application.Current.ActualThemeVariant` reflects the change

2. **Check FluentTheme dark variant application**
   - Ensure FluentTheme resources are properly loading for dark mode
   - Add explicit dark variant styles for controls that aren't theming correctly

3. **Fix invisible controls in dark mode**
   - The ComboBox dropdown icon (`DropDownGlyph`) and other icons have hardcoded colors
   - Add explicit dark variants for these elements in the styles

4. **Verify style application**
   - Ensure `:dark` pseudo-class selectors are working correctly
   - Add fallback styles with proper DynamicResource bindings to theme resources

## Implementation Steps

3. **Step 3: Add missing dark styles** - Add explicit styles for invisible controls
4. **Step 4: Verify** - Run app and test dark mode manually to confirm fix
